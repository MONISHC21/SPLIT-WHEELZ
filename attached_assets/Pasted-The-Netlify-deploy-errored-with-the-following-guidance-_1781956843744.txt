The Netlify deploy errored, with the following guidance provided:

Diagnosis
- Relevant log lines: the build runs the package script [line 59-L61](#L59-L61) and then fails because the TypeScript compiler is missing: [line 62](#L62). The npm lifecycle failure is shown at [line 63-L69](#L63-L69). Note dependencies were reported installed earlier [line 28](#L28) and the build runs from the frontend folder [line 48](#L48).
- Error type and cause: The build script executes "tsc && vite build", but tsc (the TypeScript compiler) is not available in the build environment ("sh: 1: tsc: not found" — [line 62](#L62)). This usually means TypeScript is not listed/installed in package.json (or devDependencies were not installed in the CI environment).

Solution
1. Verify TypeScript is listed in your frontend package.json
   - Open splitwheelz/frontend/package.json in your repo and confirm there is a "typescript" entry (likely under devDependencies). Example file location:
     https://github.com/MONISHC21/SPLIT-WHEELZ/blob/main/splitwheelz/frontend/package.json

2. If typescript is missing, add it and commit
   - From your project (local) run:
```bash
cd splitwheelz/frontend
npm install --save-dev typescript
git add package.json package-lock.json
git commit -m "chore: add typescript as devDependency"
git push
```
   - This ensures a local tsc binary is available at node_modules/.bin/tsc so the build script "tsc && vite build" can run.

3. Ensure a tsconfig.json exists (if your build expects it)
   - If you don't have one, create it locally:
```bash
cd splitwheelz/frontend
npx tsc --init
git add tsconfig.json
git commit -m "chore: add tsconfig"
git push
```

4. If you intentionally keep TypeScript only as a devDependency and your Netlify build is not installing devDependencies
   - Verify Netlify is installing devDependencies (it normally does). If you set NPM_CONFIG_PRODUCTION=true in your Netlify environment, devDependencies will be skipped. Either unset that env var or explicitly tell Netlify to install dev deps. You can also change the build command to force install devDeps before build:
```bash
# Example alternative build command (netlify.toml or UI)
npm ci --include=dev && npm run build
```
   - If you need docs about managing Node versions on Netlify (not required here, but useful when changing Node): https://docs.netlify.com/configure-builds/manage-dependencies/#node-js-and-javascript

5. Alternative quick fix (not recommended long-term)
   - If you don't need a separate tsc invocation and rely on Vite to handle TS, change package.json build script from:
```json
"build": "tsc && vite build"
```
   to
```json
"build": "vite build"
```
   - But only do this if you understand you will skip any type-checking performed by tsc during CI.

After making the changes, push to the repo so Netlify rebuilds. The missing "tsc" executable is the direct cause of the failure ([line 62](#L62)), and installing TypeScript (or adjusting the build script/CI settings) resolves it.
